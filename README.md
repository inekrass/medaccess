# MedAccess

Учебный проект по дисциплине UI/UX.

Тема проекта: доступный веб-интерфейс записи к врачу с панелью персонализации.

Работу выполнил: Некрасов Богдан, группа Р4150.

## Цель проекта

Цель проекта — разработать веб-прототип сервиса записи к врачу, ориентированный на доступность интерфейса. Основной акцент сделан на качестве пользовательского интерфейса, понятной навигации, поддержке клавиатурного управления и настройках отображения.

Проект не ориентирован на реализацию серверной логики, авторизации или хранения данных. Все данные используются как мок-данные на стороне фронтенда.

## Ориентация на Accessibility

MedAccess проектируется как интерфейс, которым могут пользоваться люди с разными потребностями и ограничениями. Приоритетом является не только визуальное оформление, но и возможность пройти основной сценарий записи без мыши, с увеличенным шрифтом и с использованием вспомогательных технологий.

Интерфейс должен быть понятным, последовательным и устойчивым к изменению пользовательских настроек отображения.

## Учитываемые группы пользователей

В проекте учитываются следующие группы пользователей:

- слабовидящие пользователи;
- незрячие пользователи, использующие screen reader;
- пользователи, работающие только с клавиатуры;
- пожилые пользователи;
- пользователи, которым требуется увеличенный текст, увеличенные интервалы или более заметный фокус.

## Реализуемые подходы WCAG 2.2

В проекте предусматриваются следующие подходы к доступности:

- семантическая HTML-структура с использованием `header`, `nav`, `main`, `section`, `form`, корректных заголовков и кнопок;
- skip-link для быстрого перехода к основному содержанию;
- логичный порядок фокуса при навигации с клавиатуры;
- хорошо заметная подсветка фокуса для интерактивных элементов;
- связь ошибок с полями формы через `aria-describedby`;
- явное обозначение обязательных полей;
- использование `aria-live` для важных динамических сообщений;
- понятные тексты кнопок и элементов управления;
- достаточный контраст текста и элементов интерфейса;
- адаптация интерфейса к увеличению размера шрифта и интервалов.

## Панель персонализации

В интерфейсе планируется панель персонализации со следующими настройками:

- размер шрифта: обычный, крупный, очень крупный;
- межстрочный интервал: обычный, увеличенный, большой;
- межбуквенный интервал: обычный, увеличенный, большой;
- постоянная видимая подсветка фокуса.

Выбранные настройки должны реально влиять на отображение интерфейса. Для сохранения настроек после обновления страницы используется `localStorage`.

## Основной пользовательский сценарий

Пользовательский сценарий включает следующие шаги:

1. Открытие главной страницы.
2. Настройка параметров отображения при необходимости.
3. Переход к записи к врачу.
4. Выбор специалиста или врача.
5. Выбор даты и времени приема.
6. Ввод данных пациента.
7. Исправление ошибок формы при их наличии.
8. Подтверждение записи.
9. Получение понятного сообщения об успешной записи.

## Запуск проекта

Проект использует простой стек:

- HTML;
- CSS;
- JavaScript.

Для запуска не требуются фреймворки, сборщики или серверная часть.

Способы запуска:

1. Открыть файл `index.html` в браузере.
2. Или запустить простой локальный сервер из корня проекта, например:

```bash
python3 -m http.server 8000
```

После запуска через локальный сервер проект будет доступен по адресу:

```text
http://localhost:8000
```
